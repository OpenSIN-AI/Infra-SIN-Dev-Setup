# GitHub-Profil-README

Ein Profil-README ist die `README.md`, die auf deiner GitHub-Profilseite angezeigt wird.
GitHub rendert sie automatisch, wenn das Repository exakt deinen Benutzernamen trägt.

## So richtest du es ein

1. Erstelle ein **öffentliches** Repository mit **genau deinem GitHub-Username** als Namen.
2. Lege im Root eine `README.md` an.
3. Fülle die Datei mit dem Inhalt, den du auf deinem Profil zeigen willst.
4. Nach dem Push erscheint die README automatisch auf deiner Profilseite.

## Beispiel

Wenn dein Username `steipete` ist, heißt das Repository:

```text
steipete
```

Die URL sieht dann so aus:

```text
https://github.com/steipete/steipete
```

## Minimaler Startinhalt

```md
# Hi, ich bin @deinname 👋

Ich baue Produkte, Tools und Automationen.

## Aktuell

- Arbeite an: Open Source, Webapps, Automationen
- Interessiert an: AI, Developer Tools, Produktivität

## Kontakt

- Website: https://deine-domain.example
- LinkedIn: https://linkedin.com/in/deinname
```

## Gute Inhalte

- kurze Selbstbeschreibung
- aktueller Stack oder Fokus
- Projekte / Highlights
- Kontaktmöglichkeiten
- Links zu Website, X, LinkedIn oder Portfolio

## Checkliste

- [ ] Repo-Name entspricht exakt dem Username
- [ ] Repo ist öffentlich
- [ ] `README.md` liegt im Root
- [ ] Inhalte sind aktuell und öffentlich sichtbar
